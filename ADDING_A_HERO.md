# Adding a New Hero (Student Character)

This project uses Java Swing + a simple battle engine. To add a new hero and make them show up in **CharacterSelections**, you must update **4 areas**:

1. A new `CharacterX.java` class (the hero logic)
2. Assets (the hero image)
3. `CharacterSelections.java` (so the hero can be picked)
4. `BattleScreen.java` (so the hero image can show during battle)
5. `CharacterFactory.java` (so the hero can be restored when loading a saved battle)

---

## 1) Create the new character class

### File location
Create a new file here:

`RPG_STUDENT_vs_FACULTY/src/Characters/CharacterYourHero.java`

### Template
Copy one of the existing heroes (example: `CharacterEthan.java`) and edit:

- `super("Name", "Role", "DamageType", "BestStat")`
- base stats (`maxHp`, `hp`, `maxMana`, `mana`, `morale`)
- default `position` (`"Front"`, `"Above"`, or `"Below"`)
- `getSkillname()` (3 skills)
- `useSkills(...)`
- `basicAttack(...)`
- `defend()`
- `getPassivename()` + `getPassiveValue()` (make sure arrays match in length)

Important rules:
- The hero name should match the image name mapping (see **Section 4**) and should not contain trailing spaces.
- Skills automatically get cooldown logic from the base class (3 rounds by default).

---

## 2) Add the hero image (assets)

Put the image in:

`RPG_STUDENT_vs_FACULTY/src/assets/`

Recommended naming:
- `YourHero.png`

The project loads images using `/assets/...` paths, so make sure it’s inside that folder.

---

## 3) Make the hero appear in Character Selection

### File to edit
`RPG_STUDENT_vs_FACULTY/src/Game_UI/CharacterSelections.java`

### What you need to add

#### A) Add a new button in the form (NetBeans GUI)
Because this UI is generated by NetBeans, the best way is:

1. Open the project in **NetBeans**
2. Open `CharacterSelections.form` (Design tab)
3. Drag a new `JButton` into the UI (near the other character buttons)
4. Set its icon to your new hero image:
   - Example: `/assets/YourHero.png`
5. Create an `actionPerformed` handler for it.

#### B) In the new button handler, set the selected hero
Inside the handler, do the same pattern used by the existing heroes:

```java
private void jbtnYourHeroActionPerformed(java.awt.event.ActionEvent evt) {
    currentSelectedCharacter = new CharacterYourHero();
    // Update UI preview (image/name/description) similar to existing buttons
    // Example: jlblname.setText(currentSelectedCharacter.getName());
    // Example: show skills/stats in the text areas
}
```

#### C) Ensure the “Add” button adds the selected hero
The project already uses:
- `partyMembers` list
- `parties` counter
- `currentSelectedCharacter`

So just make sure your button sets `currentSelectedCharacter` correctly and the existing “Add” logic will include it.

---

## 4) Make the hero image show in battle (and blur when dead)

### File to edit
`RPG_STUDENT_vs_FACULTY/src/Game_UI/BattleScreen.java`

There is a method that maps hero names to image paths:

```java
private String getImagePath(String characterName) {
    return switch (characterName.toLowerCase()) {
        case "shawn"    -> "/assets/Shawn.png";
        case "ethan"    -> "/assets/Ethan.png";
        case "dwight"   -> "/assets/Dwights.png";
        case "omar"     -> "/assets/Omars.png";
        case "princess" -> "/assets/Camachos.png";
        default         -> "/assets/Shawn.png";
    };
}
```

Add your hero here:

```java
case "yourhero" -> "/assets/YourHero.png";
```

Notes:
- `characterName.toLowerCase()` means you must match lowercase.
- If the name doesn’t match, the default image will be used.

---

## 5) Make the hero load correctly (Load Game / save file)

When you load a previous battle, the game rebuilds characters by name using:

`RPG_STUDENT_vs_FACULTY/src/GameEngine/CharacterFactory.java`

Add your hero to the switch:

```java
case "yourhero" -> new CharacterYourHero();
```

Important:
- The string key must match the character’s `getName()` value (lowercased).
- Avoid trailing spaces in hero names (example: `"Meruh"` is good, `"Meruh "` is bad).

---

## 5) Quick checklist (most common reasons a new hero “doesn’t show”)

- ✅ You created the new `CharacterYourHero.java` inside `src/Characters/`
- ✅ The class compiles and implements all abstract methods from `GameCharacter`
- ✅ You added an image file inside `src/assets/`
- ✅ You added a new button in `CharacterSelections.form` / `CharacterSelections.java`
- ✅ The new button sets `currentSelectedCharacter = new CharacterYourHero();`
- ✅ You updated `BattleScreen.getImagePath()` with your new hero name
- ✅ You updated `CharacterFactory.createCharacter(...)` so Load Game can restore the hero
- ✅ Your hero name has **no trailing spaces** (important!)

---

## Optional: If you want different cooldowns for that hero

Inside your hero constructor you can set custom cooldowns:

```java
setSkillCooldowns(3, 2, 5); // Skill1, Skill2, Skill3
```

By default, the project uses 3 rounds for each skill.
